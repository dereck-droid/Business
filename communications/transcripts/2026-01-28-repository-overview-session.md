## Organizational System

### Hub and Spoke Architecture

The repository uses a hub-and-spoke model where FOCUS.md serves as the daily entry point, updated every morning. README.md provides navigation to deeper documents throughout the system. The Prospect Tracker serves as an index for the entire sales pipeline. Related documents cross-reference each other to improve discoverability.

### Dynamic Priority Framework

The system employs a financial zone-based weighting system that changes decision criteria based on runway length.

In the Red Zone, when runway is less than four weeks, revenue urgency receives 50% weight in decision making. The focus is on immediate cash generation.

In the Yellow Zone, with four to eight weeks of runway, the approach is balanced with revenue receiving 35% weight. This allows for some strategic thinking while maintaining urgency.

In the Green Zone, with eight or more weeks of runway, strategic value receives 50% weight. This is when you can invest in long-term builds and partnerships.

The Prospect Scoring System ranks each opportunity by probability of close, revenue potential, days to cash, and strategic value. This creates an objective framework for prioritization.

### Communication Tracking

Every quote, meeting, and important message is logged with date stamping to prevent context loss. The naming convention of YYYY-MM-DD-prospect-name-type.md ensures chronological organization. This system ensures AI agents always have current status on all prospects and conversations.

### Document Lifecycle Management

All files include metadata headers showing date, status, and purpose. Related Documents sections create bidirectional linking between connected files. Files move to archive when deals close or projects complete. Different documents update on different frequencies: daily for FOCUS, weekly for metrics, monthly for strategy reviews.

## Why This Repository Exists

### Problem One: Context Loss

Without this system, AI agents would need to ask clarifying questions at the start of every session. Questions like "What's your current cash position?" or "Who are your top prospects?" would waste time and create friction. Now agents read FOCUS.md and understand current priorities instantly.

### Problem Two: Scattered Execution

The owner's ENFP personality excels at ideation through Wonder and evaluation through Discernment, but struggles with completion due to low Tenacity. This system provides external structure to overcome that inherent weakness. It creates accountability and tracking that doesn't depend on willpower alone.

### Problem Three: Missed Opportunities

Before this system, there was frequent confusion about "what did we quote them?" or "when did we last follow up?" Communication logs prevent this problem and ensure AI agents have current status of all prospects. Nothing falls through the cracks.
